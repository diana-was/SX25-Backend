Parked Layout Documentation
Version 1.2


Overview

The Parked.com Custom Layout feature enables additional control over the appearance of parked domains. Through the use of variables, Parked.com allow access to its content which can be used in the layout designs according to your own needs using HTML, CSS or JavaScript.

The CSS and JavaScript code can be created in the CSS and JS tabs, which automatically generate the css and js files in the layout folder and include a “link” to this files into each (landing page and resultpage) page at the time of coping the layout into the parked account.

For more complex CSS and JS or when using designs from other sources include the links to the files in the header as a normal html using {PROFILE_LINK} as the url e.g.

<link href="{PROFILE_LINK}em_v1_03.css" rel="stylesheet" type="text/css"> 
<script src="{PROFILE_LINK}jquery.min.js" type="text/javascript"></script>

Then upload the files in the “Theme Files” tab.

For images or other files used in the theme use {PROFILE_LINK} variable in the url and upload the files in the "Theme Files" tab.

Here are a few more things to keep in mind:

  • A Landing page and Results page is required
  • Required variables must be present, but optional variables may be used at your discretion
  • Keyword usage should not exceed 60 for optimimum performance

 

General Variables

The guidelines below are used to construct the general sections of the landing page and results page. The majority of these variables can be modified from within your account and will display the set content.

Variable Description Required Location Example Usage
{COPYRIGHT} Displays copyright information for current domain in the format: ©yyyy domain All rights reserved. No Both <div id="copyright">{COPYRIGHT}</div>
{DOMAIN} Displays the current domain name. No Both <div id="hdr">{DOMAIN}</div>
{FORSALE} For Sale message displayed in header of template. Displays the For Sale message as defined on the Domain Detail page. No Both <span class="forsale">{FORSALE}</span>
{LANDING_IMG} Displays the image on the landing page as defined on the Domain Detail page based on the keyword or custom content. No Landing Page <img src="{LANDING_IMG}">
{META_KEYWORDS} Meta element used to identify keywords of the document to improve the quality of search results. Displays the Meta Tags Keywords text as defined on the Domain Details page. No Both <meta name="keywords" content="{META_KEYWORDS}">
{META_TAG} Meta element used to add additional properties of the document by specifying additional meta elements. Complete meta tags should be used. Not accessible from Domain Details page. No Both <meta name="author" content="John Doe">
<meta name="date" content="2008-01-01T08:49:37+00:00">
{META_TITLE} Meta element used to identify description properties of the document. Displays the Meta Tags Description text as defined on the Domain Detail page. No Both <meta name="description" content="{META_TITLE}">
{PAGETITLE} TITLE element used to identify the contents of the document. Displays the Title Tag text as defined on the Domain Detail page. No Both <title>{PAGETITLE}</title>
{RESULTS_IMG} Displays the image on the results page as defined on the Domain Detail page based on the keyword or custom content. No Results Page <img src="{RESULTS_IMG}">
{STYLESHEET} Allows the use of Parked.com created Style Sheets. Sets the style sheet to control appearance as defined on the Domain Detail page using the Color option. No Both <link href="/style/{STYLESHEET}" rel="stylesheet" type="text/css">
{TITLE} Title used in header of template. Displays the Domain Title text as defined on the Domain Detail page. No Both <a href="/">{TITLE}</a>
{TITLE_LINK} Link used for title or logo in header of template. If specified, the link of the domain will be automatically added. No Both <a href="{TITLE_LINK}"><img src="logo.gif"/></a>
{USER_CONTENT} Displays the Custom Content as defined on the Domain Detail page. No Both <div id="userContent">{USER_CONTENT}</div>

Form Specific Variables

The guidelines below are used to construct the form section, which is located on the landing page and results page.

Variable Description Required Location Example Usage
{KEYWORDS} Displays the keyword that was used for current search. Used to populate text input field after a search. Yes Both <input type="text" name="Keywords" value="{KEYWORDS}" style="width: 200px;">
{R} Hidden form element for internal use. Yes Both <input type="hidden" name="r" value="{R}">

Additional Info

In addition to the variable requirements, the form tag and elements must be specified properly in order to function correctly. The following form tag and elements must be present and the element attributes must be specified exactly as shown:

<form method="get" action="/result.php">
	<input type="text" name="Keywords" value="{KEYWORDS}">
	<input type="hidden" name="r" value="{R}">

Specifying additional attributes, such as a class attribute, is allowed as long as additional attributes are placed after the required attributes in the example above.

The guidelines below are used to configure the keywords that appear on the landing page and results page. There are four types of keywords that can be utilized in constructing your pages. Three of the keyword types, Related, Category and Hard-coded can be used in any combination in order to meet the minimum requirement of five keywords per page. Some keyword variable types are restricted to certain pages.

Hard-coded Keyword Variables

The variables below are used to construct Hard-coded keywords, in the event you prefer not to use the provided keywords. Hard-coded keywords can be located on the landing page and results page.

Variable Description Required Location Example Usage
{R}
 Query string value for internal use. Used as part of href attribute in conjunction with user-supplied keyword and label to form <a> tag. Yes Both <a href="/result.php?Keywords=domain+parking&r={R}">Domain Parking</a>

Additional Info

The Hard-coded Keyword variable {R} is only a portion of the overall code necessary to utilize your own keywords. A user-supplied keyword and URL encoded version of that keyword are also used to form a <a> tag. The tag should be structured similar to the examples to ensure that the links function properly. A few sample Hard-coded Keyword links are shown below: 

<a href="/result.php?Keywords=domain+parking&r={R}">Domain Parking</a><br>
<a href="/result.php?Keywords=forwarding+domain+name&r={R}">Forwarding Domain Name</a><br>
<a href="/result.php?Keywords=domain+name+services&r={R}">Domain Name Services</a><br>
<a href="/result.php?Keywords=registration+service&r={R}">Registration Service</a><br>
<a href="/result.php?Keywords=name+registration+web&r={R}">Name Registration Web</a><br>

A minimum of five Related, Category or Hard-coded keywords must be included on each page in order to be considered valid.




Related Searches Keyword Variables

Guidelines below are used to construct the Related Searches section, which is located on the results page.

Variable Description Required Location Example Usage
{RELATED_SEARCHES} Related Search keywords based on keyword click or search. No Results Page <ul>{RELATED_SEARCHES}</ul>

Additional Info

Related Searches is optional, however if used it must be displayed as an unordered list. Each keyword that is returned as part of the {RELATED_SEARCHES} variable is a list item (within <li> tags), so the {RELATED_SEARCHES} should be placed within an unordered list tag (<ul>) in order to display properly. A sample structure is shown below:

<ul>
	<li>
		<a href="{RELATED_LINK}">{RELATED_SEARCH}</a>
	</li>
	<li>
		<a href="{RELATED_LINK}">{RELATED_SEARCH}</a>
	</li>
	<li>
		<a href="{RELATED_LINK}">{RELATED_SEARCH}</a>
	</li>
	<li>
		<a href="{RELATED_LINK}">{RELATED_SEARCH}</a>
	</li>
</ul>

Style sheets can be used to adjust the appearance of the list to suit your needs.

Sponsor Listings Variables

Guidelines below are used to construct the Sponsor Listings section, which is located on the results page.

External Link Variables

The guidelines below are used to configure the External Link variables which enable additional content to be added to your domains using external sources.


File Open Variable

The variable below is used to add URL-based content. The File Open Variable can appear on the landing page or results page.

Variable Description Required Location Example Usage
{FILE_OPEN}
 Link to be used for adding additional content using a URL-based file source. The source of the provided URL will replace the variable on the domain. No Both <div>{FILE_OPEN}</div>

Additional Info

The File Open section actually consist of two parts, the {FILE_OPEN} variable and the File Open URL. Both components are necessary to ensure that contents of the File Open URL are displayed properly. The {FILE_OPEN} variable in the layout and Results source code will be replaced by the contents of the File Open URL. The variables {DOMAIN} and {SEARCH_TERM} can be added to the File Open URL if you wish to pass the current domain or search term in the URL.
