SX25 Layout Documentation
Version 1.0


Overview

The SX25 Custom Layout feature enables the creation of multiple themes that can be associated with multiple domains allowing them to display their content.

Through the use of variables SX25 allows access to dynamic content for each domain which can be used in the layout designs according to your own needs.

To use CSS and JavaScript files in the layout include the links to the files in the header as a normal html using the {THEME} variable as the url e.g.

<link href="{THEME}em_v1_03.css" rel="stylesheet" type="text/css"> 
<script src="{THEME}jquery.min.js" type="text/javascript"></script>

Then upload the files in the “Theme Files” tab.

For images files used in the theme use {THEME_IMAGE} variable in the url and upload the files in the "Theme Files" tab.

Here are a few more things to keep in mind:

  • A Landing page and Results page is required
  • Required variables must be present, but optional variables may be used at your discretion
  • Keyword usage should not exceed 60 for optimimum performance

General Variables

The guidelines below are used to construct the general sections of the landing page and results page. The majority of these variables can be modified from within domain editor.

Variable Description Required Location Example Usage
{COPYRIGHT} Displays copyright information for current domain in the format: ©yyyy domain All rights reserved. No Both <div id="copyright">{COPYRIGHT}</div>
{DOMAIN} Displays the current domain name. No Both <div id="hdr">{DOMAIN}</div>
{META_KEYWORDS} Meta element used to identify keywords of the document to improve the quality of search results. Displays the Meta Tags Keywords text as defined on the Domain Details page. No Both <meta name="keywords" content="{META_KEYWORDS}">
{META_TAG} Meta element used to add additional properties of the document by specifying additional meta elements. Complete meta tags should be used. Not accessible from Domain Details page. No Both <meta name="author" content="John Doe">
<meta name="date" content="2008-01-01T08:49:37+00:00">
{META_TITLE} Meta element used to identify description properties of the document. Displays the Meta Tags Description text as defined on the Domain Detail page. No Both <meta name="description" content="{META_TITLE}">
{PAGETITLE} TITLE element used to identify the contents of the document. Displays the Title Tag text as defined on the Domain Detail page. No Both <title>{PAGETITLE}</title>
{RESULTS_IMG} Displays the image on the results page as defined on the Domain Detail page based on the keyword or custom content. No Results Page <img src="{RESULTS_IMG}">
{LANDING_IMG} Displays the image on the landing page as defined on the Domain Detail page based on the keyword or custom content. No Landing Page <img src="{LANDING_IMG}">
{TITLE} Title used in header of template. Displays the Domain Title text as defined on the Domain Detail page. No Both <a href="/">{TITLE}</a>
{TITLE_LINK} Link used for title or logo in header of template. If specified, the link of the domain will be automatically added. No Both <a href="{TITLE_LINK}"><img src="logo.gif"/></a>
{DOMAIN_ID} The id of the domain in the domain table. No Both <a href="display.php?id={DOMAIN_ID}"><img src="logo.gif"/></a>
{REFERER_PAGE} The referer site. No Both <span >{REFERER_PAGE}</span>
{HOME} The complete url to the root directory. No Both <a href="{HOME}"><img src="{HOME}index.php"/></a>
{THEME} The complete url to the theme directory. No Both <script src="{THEME}jquery.min.js" type="text/javascript"></script>
{THEME_IMAGE} The complete url image directory inside the theme directory. No Both <img src="{THEME_IMAGE}logo.gif"/>
{IMAGE_LIBRARY} The complete url to image library. No Both <a href="example.php"><img src="{IMAGE_LIBRARY}logo.gif"/></a>
{CSS_LIBRARY} The complete url to css library for generation Z layouts. No Both <link rel="stylesheet" type="text/css" href="{CSS_LIBRARY}{DOMAIN}.css"/>

Keyword Variables

The guidelines below are used to configure the keywords that appear on the landing page and results page. There are two types of keywords that can be utilized in constructing your pages.

Variable Description Required Location Example Usage
{KEYWORDS} Displays the keyword that was used for current search sended in request as "k=". Used to populate text input field after a search. Yes Both <input type="text" name="Keywords" value="{KEYWORDS}" style="width: 200px;">
{MAPPING_KEYWORD} Displays the keyword that was used for current search sended in request as "keyword=" or "keywords=". Used to populate text input field after a search. No Both <input type="text" name="Keywords" value="{MAPPING_KEYWORD}" style="width: 200px;">
{DOMAIN_KEYWORD} Displays the keyword of the domain. No Both <input type="text" name="Keywords" value="{DOMAIN_KEYWORD}" style="width: 200px;">
{SEARCH_TERM} Displays the keyword that was used for current search sended in request as "k=". Used to populate text input field after a search. No Both <input type="text" name="Keywords" value="{SEARCH_TERM}" style="width: 200px;">
{KEYWORD_URL} Displays the keyword that was used for current search sended in request as "k=" URL encoded. No Both <a href="display.php?keyword={KEYWORD_URL}"><img src="logo.gif"/></a>

Additional Info

The Sponsor Listings section actually consist of two parts, the {SPONSOR_LISTINGS} or {SPONSOR_LISTINGS_x} variables and the Sponsored Listings Source Code. Both components are necessary to ensure that Sponsor Listings are displayed properly. The {SPONSOR_LISTINGS} variable in the layout source code will be replaced by the Sponsored Listings Source Code. Using the source code allows more flexibility in customizing the Sponsor Listings to suit your needs. A sample Sponsored Listings Source Code structure is shown below:

<p>

   <a class="URLtext" href="{CLICK_LINK}" target='_blank'>{DESCRIPTION}</a><br>
   <a class="URLbottom" href="{CLICK_LINK}" target='_blank'>{SITEHOST}</a>
</p>

Additional Info

The static pages in the site are linked as domainName.com/pageName.php e.g: webinfo.net/contact_us.php. This pages will be resolve by the application searching in the database for a page with name "contac_us" and replace the {PAGE_TITLE} and {PAGE_CONTENT} with the page information.
The code between {PAGE_REPLACE_BEGIN} and {PAGE_REPLACE_END} will be deleted form the displayed page.
